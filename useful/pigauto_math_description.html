1. Overview & design principles

pigauto fits a per-trait blend of two predictors:

$$\widehat{y}_{ij} \;=\; \underbrace{(1 - r_j) \cdot \mu_{ij}^{\text{baseline}}}_{\text{phylogenetic prior}} \;+\; \underbrace{r_j \cdot \delta_{ij}^{\text{GNN}}}_{\text{neural correction}} \;+\; \underbrace{\mathrm{X}_i \widehat\beta_j^{\text{(linear)}}}_{\text{Fix C/D residual stream}}$$

where

  • $i$ indexes observations (= species in single-obs mode; = trait measurements in multi-obs mode);
  • $j$ indexes latent trait columns (continuous traits use one latent column; categorical with $K$ classes uses $K$);
  • $\mu^{\text{baseline}}$ is computed analytically (no training) — see §5–§6;
  • $\delta^{\text{GNN}}$ is the output of a graph transformer trained by stochastic gradient descent;
  • $r_j \in (0, r_{\max})$ is a per-latent-column gate calibrated on a held-out validation split;
  • $\widehat\beta_j$ is a small linear-on-covariates path that contributes outside the gate.

Three properties are designed in:

  1. Safety floor. $r_j = 0$ is always a valid fallback. When the calibrated gate closes, the model collapses to the analytical baseline. Users passing covariates that are uninformative beyond phylogeny pay no accuracy penalty.
  2. Mixed types from a single API. Continuous, count, ordinal, binary, categorical, proportion, zero-inflated count, and multi-proportion (compositional) traits coexist in a single latent matrix; the loss switches between MSE / BCE / cross-entropy per trait type.
  3. Uncertainty quantification. Conformal prediction intervals (distribution-free, calibrated on a held-out split) and multiple imputation (MC-dropout or conformal-score sampling) are integrated.

4. Phylogenetic graph construction

From the tree $T$, build_phylo_graph() produces three pieces:

$$\mathbf{D}_{kl} = \mathrm{cophenetic}(T)_{kl}, \quad \mathbf{A}_{kl} = \exp\!\left(-\frac{\mathbf{D}_{kl}^2}{2\sigma^2}\right)$$

$\sigma$ is set adaptively from the median pairwise distance. The Laplacian $\mathbf{L} = \mathbf{D}_{\text{deg}} - \mathbf{A}$ is eigendecomposed and the top $k$ eigenvectors form $\mathbf{C}$:

$$\mathbf{L} \mathbf{C} = \mathbf{C} \boldsymbol\Lambda, \quad \mathbf{C} \in \mathbb{R}^{n_s \times k}$$

$k$ scales adaptively with $n_s$ (default: $k = 4$ at $n_s = 30$, $k = 32$ at $n_s = 640$+). The squared distance matrix $\mathbf{D}^2$ is also passed downstream for rate-aware attention bandwidth (see §7).

5. BM baseline — conditional multivariate normal

For each continuous (BM-eligible) latent column $j$, observed cells $\mathcal{O}_j$ and missing cells $\mathcal{H}_j$. Under the standard Brownian-motion model with phylogenetic correlation $R$:

$$\mathbf{X}_{:,j} \;\sim\; \mathcal{N}(\mu_j \mathbf{1}, \;\sigma_j^2 \cdot R)$$

The conditional mean and variance for missing cells follow the standard kriging formulas. Let $R_{oo} = R[\mathcal{O}_j, \mathcal{O}_j]$ and $R_{ho} = R[\mathcal{H}_j, \mathcal{O}_j]$. With Cholesky $R_{oo} = L L^\top$:

$$\widehat\mu_j = \frac{\mathbf{1}^\top R_{oo}^{-1} \mathbf{X}_{\mathcal{O}_j, j}}{\mathbf{1}^\top R_{oo}^{-1} \mathbf{1}}, \quad \widehat\sigma_j^2 = \frac{(\mathbf{X}_{\mathcal{O}_j,j} - \widehat\mu_j)^\top R_{oo}^{-1} (\mathbf{X}_{\mathcal{O}_j,j} - \widehat\mu_j)}{n_o - 1}$$

$$\boxed{\;\mu^{\text{BM}}_{\mathcal{H}_j, j} = \widehat\mu_j + R_{ho} R_{oo}^{-1} (\mathbf{X}_{\mathcal{O}_j, j} - \widehat\mu_j)\;}$$

$$\sigma^{\text{BM}}_{\mathcal{H}_j, j} = \widehat\sigma_j \sqrt{1 - \mathrm{diag}(R_{ho} R_{oo}^{-1} R_{ho}^\top)}$$

This is computed analytically once before training; it does not depend on the GNN. Implementation in R/bm_internal.R::bm_impute_col(). Numerical stability uses a backed-off Cholesky nugget on $R_{oo}$.

For binary and categorical traits, a phylogenetic label propagation baseline replaces the conditional MVN: the per-species probability is the kernel-weighted mean of observed labels using $\mathbf{A}$ as similarity. (Categorical with $K$ classes uses one-vs-rest threshold-joint baselines via Rphylopars; see R/ovr_categorical.R.)

6. Covariate-aware baseline (Fix G) — GLS regression

When the user passes covariates $\mathbf{Z}$ (an $n_s \times d_z$ matrix), the BM baseline is upgraded to a phylogenetic GLS regression with covariate fixed effects. Let $\mathbf{X}_{\text{des}} = [\mathbf{1}, \mathbf{Z}]$ be the design matrix.

$$\mathbf{y}_{:,j} \;=\; \mathbf{X}_{\text{des}} \boldsymbol\beta_j + \mathbf{u}_j, \quad \mathbf{u}_j \sim \mathcal{N}(\mathbf{0}, \sigma_j^2 R)$$

The GLS estimate is

$$\widehat{\boldsymbol\beta}_j = \left( \mathbf{X}_{\text{des},o}^\top R_{oo}^{-1} \mathbf{X}_{\text{des},o} + \lambda_{\text{ridge}} \mathbf{I}_{\text{non-int}} \right)^{-1} \mathbf{X}_{\text{des},o}^\top R_{oo}^{-1} \mathbf{X}_{\mathcal{O}_j, j}$$

(default $\lambda_{\text{ridge}} = 0$; $\mathbf{I}_{\text{non-int}}$ has 0 on the intercept diagonal). The BLUP for held-out cells combines the fixed-effects fit with the BM-correlated residual:

$$\boxed{\;\mu^{\text{cov}}_{\mathcal{H}_j, j} = \mathbf{X}_{\text{des},h} \widehat{\boldsymbol\beta}_j \;+\; R_{ho} R_{oo}^{-1} \big( \mathbf{X}_{\mathcal{O}_j,j} - \mathbf{X}_{\text{des},o} \widehat{\boldsymbol\beta}_j \big)\;}$$

This is mathematically identical to phylolm(model="BM")'s prediction (unit-tested to $10^{-7}$).

The likelihood-ratio gate

To guard against fitting noise on phylo-redundant covariates, a one-line check decides whether to use the cov-aware fit or fall back to the cov-blind BM baseline:

$$\text{relative reduction} = \frac{\widehat\sigma^2_{\text{intercept-only}} - \widehat\sigma^2_{\text{cov-aware}}}{\widehat\sigma^2_{\text{intercept-only}}}$$

If the relative reduction does not exceed lrt_threshold (default $0.02$), used_cov is set FALSE and the intercept-only GLS solution is used. This was decisive on Delhey n=5,809 where climate is genuinely phylo-redundant: the LRT correctly detects no signal and falls back, preventing the GNN from being pulled toward a noisy "all-zero coefficients" baseline.

7. The GNN — graph transformer architecture

The neural correction $\delta$ is computed by a graph transformer (R/model_residual_dae.R::ResidualPhyloDAE) with the following data flow:

INPUT x (n × p_ℓ) corrupted trait input (NA cells → learned mask token) C_obs (n × k) spectral phylo features (broadcast obs ← species in multi-obs) covs (n × d_cov) cov_dim = p_ℓ + 1 + d_z [baseline_mu | NA-mask indicator | user covariates] ENCODER (2-layer MLP) combined = concat(x, C_obs, covs) h = enc1(combined) → ReLU → dropout → enc2 → ReLU ∈ ℝ^(n × h) POOL TO SPECIES (multi-obs only) h_species = scatter_mean(h, obs_to_species) ∈ ℝ^(n_s × h) GNN STACK (n_gnn_layers Graph Transformer Blocks) if n_user_cov > 0: cov_h = cov_encoder(user_covs) (Fix B) cov_h_sp = scatter_mean(cov_h, obs_to_species) if multi_obs else cov_h for ℓ in 1..n_gnn_layers: h_species = TransformerBlock_ℓ(h_species, A, D², cov_h_sp) (Fix H) -- multi-head self-attn with phylo-Gaussian bias -- pre-norm, FFN, residual -- Fix H: residual injection of cov_h_sp at each layer BROADCAST BACK (multi-obs only) h = h_species[obs_to_species] ∈ ℝ^(n × h) COVARIATE REFINEMENT (Fix B — fires when n_user_cov > 0) cov_h = cov_encoder(user_covs) # (n × h) h = h + obs_refine(concat(h, cov_h)) # residual DECODER (2-layer MLP) h = dec1(h) → ReLU δ = dec2(h) ∈ ℝ^(n × p_ℓ) LINEAR COV HEAD (Fix C/D — fires when n_user_cov > 0) fixed_effects = cov_linear(user_covs) ∈ ℝ^(n × p_ℓ) # initialised to small weights × 0.01 for safety

The Graph Transformer Block

Each transformer block (R/graph_transformer_block.R) computes:

$$\mathrm{attn\_scores}_{h,kl} = \frac{(W_Q^h h_k)^\top (W_K^h h_l)}{\sqrt{d_{\text{head}}}} + b_{h,kl}^{\text{phylo}}$$

The phylogenetic prior bias has two modes. The rate-aware mode (B2) uses per-head learned Gaussian bandwidth $\beta_h$ on squared cophenetic distance:

$$b_{h,kl}^{\text{phylo}} \;=\; -\frac{\mathbf{D}^2_{kl}}{2 \cdot \mathrm{softplus}(\beta_h)^2}$$

so each head learns its own phylogenetic scale (some heads tight = close relatives, others broad = distant relatives). The legacy mode uses a learned scaling on $\log \mathbf{A}$. Output is the standard transformer block:

$$h^{\text{(out)}} = h + W_O \cdot \mathrm{softmax}(\mathrm{attn\_scores}) V + \mathrm{FFN}(\mathrm{LN}(h^{\text{post-attn}})) + \mathrm{cov\_inject}(\mathrm{cov\_h})$$

where the FFN output projection and (Fix H) cov_inject linear weights are both initialised to zero so each block $\approx$ identity at training step 0. This preserves the gate-closed-at-init safety property: an untrained pigauto = the analytical baseline.

8. The blended prediction

Putting it together, the per-cell prediction is

$$\boxed{\;\widehat{y}_{ij} \;=\; (1 - r_j) \cdot \mu^{\text{baseline}}_{ij} \;+\; r_j \cdot \delta^{\text{GNN}}_{ij} \;+\; (\mathbf{Z}_i \widehat\beta_j^{\text{(linear)}})\;}$$

where the gate is

$$r_j \;=\; \sigma(\rho_j) \cdot r_{\max}, \qquad \rho_j \in \mathbb{R}, \quad r_{\max} \in (0, 1]$$

$r_{\max}$ (default 0.8) caps the gate so the baseline always gets at least 20% weight even if the GNN tries to dominate. The per-latent-column vector $\boldsymbol\rho$ initialises at $\rho_j = -1$ for continuous types ($r_j \approx 0.135 \cdot r_{\max}$) and near 0 for binary/categorical (gate near $r_{\max}/2$).

9. Loss function and training

The model is trained as a denoising autoencoder. At each epoch, a random subset of observed cells is corrupted (replaced with the learned mask token) at corruption rate $\tau$ that ramps from $\tau_0 = 0.20$ to $\tau_1 = 0.55$ over corruption_ramp epochs.

The loss is a weighted sum of three terms:

$$\mathcal{L} \;=\; \mathcal{L}_{\text{main}} \;+\; \lambda_{\text{shrink}} \cdot \mathrm{MSE}(\delta - \mu^{\text{baseline}}) \;+\; \lambda_{\text{gate}} \cdot \mathrm{MSE}(r)$$

Each term:

TermFormulaDefault weightPurpose
$\mathcal{L}_{\text{main}}$type-appropriate (MSE / BCE / CE per trait)1match the blended prediction to the truth
shrinkage$\frac{1}{n p_\ell} \sum_{ij} (\delta_{ij} - \mu^{\text{baseline}}_{ij})^2$0.03prefer $\delta \approx$ baseline when no signal supports a deviation
gate$\frac{1}{p_\ell} \sum_j r_j^2$0.01prefer closed gate when no signal supports opening

Optimiser is AdamW with linear-decay learning rate (default base 0.005, warmup 200 epochs). Default total epochs is 3,000 with patience-based early stopping at 100 epochs of no validation improvement.

10. Calibration and safety floor

After training, the gate $r_j$ is recalibrated per latent column on a held-out validation split. Let $\mathcal{V}_j$ be the validation cells for trait $j$. The calibrated gate is

$$r_j^{\text{cal}} = \arg\min_{r \in [0, r_{\max}]} \; \frac{1}{|\mathcal{V}_j|} \sum_{i \in \mathcal{V}_j} \ell\big( (1-r) \mu_{ij}^{\text{baseline}} + r \delta_{ij}^{\text{GNN}}, \; y_{ij} \big)$$

where $\ell$ is the type-appropriate per-cell loss (squared error for continuous, 0–1 loss for discrete with a cell-count floor). The grid is fine: 41 candidate values uniformly between 0 and $r_{\max}$.

The safety floor

When safety_floor = TRUE (default), $r_j^{\text{cal}}$ is constrained so that the calibrated blend's val RMSE is no worse than the column-mean prediction's val RMSE:

$$r_j^{\text{cal}} \;=\; \arg\min_r \; \mathrm{val\_RMSE}(r) \quad \text{s.t.} \quad \mathrm{val\_RMSE}(r) \le \mathrm{val\_RMSE}_{\text{column-mean}}$$

If no $r$ satisfies the constraint (rare; happens when the GNN is degraded), the gate is forced to $r=0$ (pure baseline). This gives the user a guarantee that pigauto's predictions are no worse than column-mean, by construction.

11. Multiple imputation and uncertainty

pigauto produces three distinct uncertainty quantifications. Each has different validity and downstream uses:

11.1 Baseline SE (analytic)

From the conditional MVN math: $\sigma^{\text{BM}}_{ij} = \widehat\sigma_j \sqrt{1 - h_{ij}}$ where $h_{ij}$ is the leverage of cell $ij$. Exact under BM; model-dependent.

11.2 Conformal prediction intervals (distribution-free)

After training, on the held-out validation set, compute conformal scores

$$s_j = \mathrm{quantile}\big( \big| \mathbf{y}_{\mathcal{V}_j} - \widehat{\mathbf{y}}_{\mathcal{V}_j} \big|, \; \lceil (1-\alpha)(1+1/|\mathcal{V}_j|) \rceil / |\mathcal{V}_j| \big)$$

The 95 % prediction interval for held-out cell $i,j$ is $\widehat y_{ij} \pm s_j$. By the split-conformal theorem this has marginal coverage exactly $\ge 1 - \alpha$, regardless of the model's correctness.

11.3 Multiple imputation draws

Two methods selectable via draws_method:

  • conformal (default): single forward pass; held-out cells sampled from $\mathcal{N}(\widehat y_{ij}, s_j / 1.96)$ on the latent scale (or Bernoulli/categorical from the probability vector for discrete types).
  • MC dropout: $M$ forward passes with dropout active. Each imputation $m$ first draws a baseline noise instance $t^{\text{BM}} \sim \mathcal{N}(\mu^{\text{BM}}, (\sigma^{\text{BM}})^2)$ on the latent scale, then blends with the dropout GNN output.

For Rubin's-rules pooling of downstream models, the conformal method is preferred because conformal scores are calibrated against actual held-out residuals regardless of gate value.

12. Mixed-type handling

All trait types live in a single latent matrix $\mathbf{X}$. The GNN delta $\delta$ has the same dimensionality. The differences are at decoding and loss:

TypeDecodingLoss
continuousinverse z + (inverse log if applied)$\mathrm{MSE}$
binary$\sigma(\widehat y_{ij})$ → class probability$\mathrm{BCE}(\sigma(\widehat y), \, y)$
categorical ($K$)$\mathrm{softmax}$ over the $K$ latent columns$\mathrm{CE}(\mathrm{softmax}, \, y)$
ordinalround $z^{-1}(\widehat y)$ to nearest class$\mathrm{MSE}$ (z-scored class index)
multi-proportion ($K$ cols summing to 1)per-row $\mathrm{softmax}$ on CLR$\mathrm{MSE}$ on CLR latent

The dispatcher in compute_mixed_loss() applies each loss to its trait's latent columns and sums them with equal weight. Per-trait reweighting is possible via the trait_loss_weights argument.

13. Multi-observation mode

When multiple trait measurements per species are available (e.g. CTmax measured at several acclimation temperatures), pigauto operates in multi-obs mode. Set via preprocess_traits(traits, tree, species_col = "species").

The matrix $\mathbf{X}$ has $n > n_s$ rows, and an obs_to_species long tensor maps each observation to its species index. The GNN forward is

  1. Encode at observation level: $h = \mathrm{enc}(\mathbf{X}, \mathbf{C}_{\text{obs}}, \mathbf{Z})$.
  2. Aggregate to species: $h_{\text{sp}} = \mathrm{scatter\_mean}(h, \mathrm{obs\_to\_species})$.
  3. Message-pass at species level on $\mathbf{A}$.
  4. Broadcast back: $h \leftarrow h_{\text{sp}}[\mathrm{obs\_to\_species}]$.
  5. Refine: $h \leftarrow h + \mathrm{obs\_refine}(\mathrm{concat}(h, \mathrm{cov\_h}))$ — restores within-species covariate variation lost during pooling.
  6. Decode to per-observation $\delta$.

The baseline $\mu^{\text{baseline}}$ is computed at species level (using species-mean of observations as the "observation" for the conditional MVN), then expanded to obs level via obs_to_species.

14. End-to-end algorithm pseudocode

function impute(traits, tree, covariates = NULL):
    # Step 1: preprocess
    pd  <- preprocess_traits(traits, tree, covariates)
    spl <- make_missing_splits(pd, val_frac = 0.5, miss_frac = 0.20)

    # Step 2: phylogenetic graph
    g <- build_phylo_graph(tree, k_eigen = "auto")

    # Step 3: analytical baseline
    bl <- fit_baseline(pd, tree, splits = spl, graph = g)
        # for each trait j:
        #   if type = continuous AND covariates supplied:
        #       (μ_j, σ_j) ← bm_impute_col_with_cov(X[,j], [1, Z], R)
        #                    [GLS regression + LRT gate]
        #   elif type = continuous:
        #       (μ_j, σ_j) ← bm_impute_col(X[,j], R)
        #   elif type = binary or categorical:
        #       (μ_j, σ_j) ← phylogenetic label propagation
        #   etc.

    # Step 4: train the GNN
    fit <- fit_pigauto(pd, tree, splits = spl, graph = g, baseline = bl,
                       covariates = pd$covariates,
                       epochs = 3000, optimiser = AdamW)

        for epoch in 1..epochs:
            # 4a. corrupt observed cells (denoising autoencoder objective)
            X_corrupted <- mask random τ-fraction of observed cells with mask_token

            # 4b. forward pass
            δ, r <- model(X_corrupted, C, covs, A, D², obs_to_species)
            ŷ    <- (1 - r) * μ_baseline + r * δ + cov_linear(Z)

            # 4c. mixed-type loss + regularizers
            L_main   <- compute_mixed_loss(ŷ, y_truth, trait_map)
            L_shrink <- mean((δ - μ_baseline)^2)
            L_gate   <- mean(r^2)
            L <- L_main + λ_shrink * L_shrink + λ_gate * L_gate

            # 4d. backward + step
            L.backward(); optimiser.step()

            # 4e. early stopping check on val cells
            if patience exceeded: break

    # Step 5: per-trait gate calibration on val
    for j in 1..p_ℓ:
        r_cal[j] <- argmin_{r in [0, r_max]} val_RMSE_j(r)
        if safety_floor: enforce val_RMSE_j(r_cal[j]) ≤ val_RMSE_column_mean_j

    # Step 6: conformal scores
    s_j <- quantile(|y_val - ŷ_val|, 1-α)  per trait j

    # Step 7: predict held-out cells
    δ_pred, _ <- model.eval(X_full, ...)   # eval mode, no dropout
    ŷ_pred   <- (1 - r_cal) * μ_baseline + r_cal * δ_pred + cov_linear(Z)

    # Step 8: multiple imputation
    if n_imputations > 1:
        for m in 1..n_imputations:
            sample y_im[m] from N(ŷ_pred, s/1.96)   [conformal method]
              -- or MC dropout draws of δ + BM-noise draws of μ_baseline

    # Step 9: decode latent → user trait scale
    completed <- decode(ŷ_pred, trait_map)

    return list(completed = completed, prediction = list(
        imputed = ŷ_pred, conformal_lower = ŷ_pred - 1.96*s,
        conformal_upper = ŷ_pred + 1.96*s, imputed_datasets = y_im),
        fit = fit, baseline = bl, splits = spl)
