# Safety floor via three-way mean-gate — design spec

**Status:** approved (brainstorming 2026-04-23).
**Branch:** `feature/safety-floor-mean-gate`.
**Plan:** `plans/2026-04-23-safety-floor-mean-gate.md` (to be written next).
**Successor spec:** plant environmental covariates (to follow when this lands clean).

---

## 1. Goal

Make pigauto's prediction **never worse than the grand-mean baseline on
validation, by construction**, and verify the same property holds on
held-out test (within a 2 % sampling-noise tolerance) on five taxa before
each PR. Deliver as a default behaviour in v0.9.1.9002-dev.

## 2. Background

The plants bench (`bench_bien.R`, 2026-04-23, n=4,745,
`n_imputations=20`) showed pigauto 15–101 % worse than the grand-mean
baseline on `height_m`, `leaf_area`, `seed_mass`, `sla`, with only
`wood_density` lifting (−6 %). Commit `16ce036` documents this as an
honest boundary-case finding; root cause is weak phylogenetic signal
(pooled BIEN species means + random within-genus polytomy resolution
in V.PhyloMaker2 S3), and pigauto's calibrated gate correctly closes
to zero on weak-signal traits — but the fallback is then the BM
baseline, which is itself weaker than the grand mean on those traits.

The current calibration is a two-way grid search:
`pred = (1 - r) · BM + r · GNN`, `r ∈ [0, 1]`. The grand mean is not in
the search space, so on plants the calibrator picks `r = 0` (pure BM)
and pigauto inherits the weak BM's loss relative to mean.

The proposed fix extends calibration to a three-way convex combination
`pred = r_BM · BM + r_GNN · GNN + r_MEAN · MEAN` on the simplex
`r_BM + r_GNN + r_MEAN = 1`, so the grand mean is always in the search
space. The training loop and torch model are untouched; `r_MEAN` is a
post-training calibration knob only.

**Paper claim this unlocks.** "pigauto is never worse than the grand
mean on validation by construction, and within 2 % of mean on held-out
test across birds, mammals, fish, amphibians, and plants."

## 3. Non-goals

This spec does **not** cover:

- Environmental covariates (bioclim / SoilGrids / GBIF) for plants →
  separate spec, next in the queue.
- Per-trait phylogenetic-signal gate (Pagel's λ / Blomberg's K
  thresholding) → separate spec, after this lands clean. The safety
  floor alone is the minimum viable fix.
- Architecture changes (trait-similarity attention, foundation-model
  pretraining) → follow-up work after the covariate spec ships.
- Changes to the BM baseline, GNN model, training loop, training-time
  gate (`res_raw` sigmoid), conformal scores, or MC-dropout draw
  machinery. All of those stay identical.

## 4. Architecture

### 4.1 The blend formula

**Before:**

```
pred_ij = (1 - r_j) · BM_baseline_ij + r_j · GNN_delta_ij
```

where `r_j` is a per-latent-column scalar in `[0, 1]` from the
post-training calibration grid.

**After:**

```
pred_ij = r_BM_j · BM_baseline_ij + r_GNN_j · GNN_delta_ij + r_MEAN_j · MEAN_j
```

where `(r_BM_j, r_GNN_j, r_MEAN_j)` is a 3-vector on the simplex
and `MEAN_j` is a scalar computed once per latent column from the
training observations.

### 4.2 The simplex grid

231 candidate points generated by
`simplex_grid(step = 0.05)`:

```r
simplex_grid <- function(step = 0.05) {
  # Returns a matrix with 3 columns (r_bm, r_gnn, r_mean) and
  # (1 / step + 1 choose 2) rows. For step = 0.05: 231 rows.
  k    <- as.integer(round(1 / step))
  out  <- matrix(NA_real_, nrow = (k + 1) * (k + 2) / 2L, ncol = 3L)
  row  <- 1L
  for (i in 0:k) for (j in 0:(k - i)) {
    out[row, ] <- c(i, j, k - i - j) / k
    row <- row + 1L
  }
  colnames(out) <- c("r_bm", "r_gnn", "r_mean")
  out
}
```

Each row satisfies `sum(row) == 1` exactly (integer arithmetic before
division).

### 4.3 Calibration procedure

Replace the 1-D calibration block at the end of `fit_pigauto.R` with:

```r
calibrate_three_way <- function(bm_val, gnn_val, truth_val, mean_j,
                                  is_continuous,
                                  grid = simplex_grid(0.05)) {
  # All vectors on the LATENT scale (z-scored for continuous/count/
  # ordinal/proportion; logit-scale for binary/categorical). Because
  # z-scoring and logit-ing are linear, argmin of MSE on latent scale
  # equals argmin of MSE on raw scale. The invariant
  # `pigauto_latent_RMSE <= mean_latent_RMSE` transforms to the raw
  # scale equivalent automatically.
  #
  # bm_val, gnn_val: length-n_val vectors from validation-split predict
  # truth_val:       length-n_val vector of held-out truth on this col
  # mean_j:          scalar grand mean of the TRAINING observations on
  #                  this latent column. On z-scored continuous cols
  #                  this is ~0 (not exactly 0 because z-scoring uses
  #                  all preprocessed observations, including ones that
  #                  later become held-out test/val). On logit-scale
  #                  binary cols this is `qlogis(clip(p_class1, 0.01, 0.99))`
  #                  where p_class1 is the training class-1 frequency.
  # is_continuous:   logical — continuous/count/ordinal/proportion/zi_mag
  #                  use MSE; binary/categorical use 0-1 loss
  stopifnot(length(bm_val) == length(truth_val),
            length(gnn_val) == length(truth_val))
  loss_fn <- if (is_continuous)
    function(p) mean((truth_val - p)^2, na.rm = TRUE)
  else
    function(p) mean(round(p) != round(truth_val), na.rm = TRUE)
  best_loss <- Inf
  best_w    <- c(1, 0, 0)   # pure BM as tie-breaker default
  for (i in seq_len(nrow(grid))) {
    w  <- grid[i, ]
    p  <- w[1] * bm_val + w[2] * gnn_val + w[3] * mean_j
    lo <- loss_fn(p)
    if (is.finite(lo) && lo < best_loss) {
      best_loss <- lo
      best_w    <- w
    }
  }
  list(r_bm = best_w[1], r_gnn = best_w[2], r_mean = best_w[3],
        loss = best_loss)
}
```

**Invariant (by construction):** because `(0, 0, 1)` is in the grid,
`best_loss ≤ loss_fn(mean_j)` for every column, exactly.

### 4.4 Storage in `pigauto_fit`

The fit object gains four new named slots alongside the existing ones:

```r
fit$r_cal_bm            # named numeric, length p_latent
fit$r_cal_gnn           # named numeric, length p_latent
fit$r_cal_mean          # named numeric, length p_latent
fit$mean_baseline_per_col  # named numeric, length p_latent
```

Names come from `colnames(data$X_scaled)` (the latent names,
including K sub-columns per categorical trait).

For **backward compatibility**, the legacy slot `fit$r_cal` is still
written on fits produced with `safety_floor = TRUE`, set to
`fit$r_cal_gnn` (the closest equivalent). Any downstream user code
reading `fit$r_cal` keeps working, but will under-describe the blend
— a warning is emitted once per session if read from a
`safety_floor = TRUE` fit.

### 4.5 Prediction

In `predict.pigauto_fit()`, replace the blend line with:

```r
# Backward-compat: old saved fits lack r_cal_mean
rbm  <- fit$r_cal_bm   %||% (1 - fit$r_cal)
rgnn <- fit$r_cal_gnn  %||% fit$r_cal
rmn  <- fit$r_cal_mean %||% rep(0, length(rbm))
mu   <- fit$mean_baseline_per_col %||% rep(0, length(rbm))

# Broadcast over observations
pred <- sweep(BM, 2L, rbm,  `*`) +
         sweep(GNN, 2L, rgnn, `*`) +
         matrix(rep(rmn * mu, each = nrow(BM)), nrow = nrow(BM))
```

### 4.6 Multiple-imputation draw handling

**`multi_impute(draws_method = "conformal")`** — unchanged
in logic. The draw center is the new 3-way blend (which `predict()`
already returns), and the conformal score is still calibrated on the
blended residuals. No code change needed in `multi_impute.R`; the
mean term propagates automatically through `predict()`.

**`multi_impute(draws_method = "mc_dropout")`** — per-draw the
existing BM-draw + GNN-dropout flow stays intact; add the mean term:

```r
# per-draw m
t_bm_draw <- rnorm(n, BM_mu, BM_se)         # unchanged
pred_m    <- r_bm   * t_bm_draw +
              r_gnn  * GNN_dropout(t_bm_draw, covs) +
              r_mean * MEAN   # the mean term is deterministic per draw
```

The mean term contributes no between-draw variance (it is a scalar per
column), so existing Rubin-pooled SE calculations remain correct.

**`multi_impute_trees(share_gnn = TRUE)`** — the `r_cal_*` weights and
`mean_baseline_per_col` are post-training artifacts on the single
reference-tree `pigauto_fit`; they are reused verbatim across posterior
trees, exactly as `r_cal` is today. When `predict_on_new_tree()`
recomputes the BM baseline on a new tree, the mean term stays fixed
(it is a property of the training traits, not the tree).

### 4.7 Discrete traits

For binary, categorical, and ordinal traits the same simplex grid is
searched with 0-1 loss. The blend math is always on the latent (log-odds
for binary/categorical, z-score for ordinal) scale, consistent with how
`bm_val` and `gnn_val` are produced by `fit_baseline()` and
`predict_gnn()`. `MEAN_j` is:

- **binary**: `qlogis(clip(p_class1, 0.01, 0.99))` where `p_class1` is
  the training class-1 frequency. This is a scalar in logit space,
  matching the scale of `bm_val` (label-propagation or threshold-joint
  baseline, also logit-scale). Decoded via `plogis()` to a probability
  for the 0-1 loss evaluation.
- **categorical**: one scalar per class-indicator sub-column.
  `MEAN_j_k = qlogis(clip(mean(Y_k == 1), 0.01, 0.99))`. The per-column
  blend produces a length-K logit vector that is softmax'd to a class
  distribution, exactly as today.
- **ordinal**: the z-scored median integer class over training
  observations (same z-scoring used by `trait_map[[j]]`).
- **count / zi_count magnitude / proportion**: as for continuous —
  `mean_j` on the z-scored log1p / logit / identity latent scale.

The clip to `[0.01, 0.99]` before `qlogis()` matches the bound used
elsewhere in pigauto to avoid `±Inf` mean terms on degenerate traits
(observed class-1 frequency 0 or 1). Tested in smoke canary item 8
(added).

### 4.8 New API surface

```r
# R/impute.R
impute(traits, tree, ...,
       safety_floor = TRUE,      # NEW, default TRUE
       ...)

# R/fit_pigauto.R
fit_pigauto(data, baseline, ...,
            safety_floor = TRUE, # NEW, default TRUE
            ...)
```

When `safety_floor = FALSE`, the 1-D calibration is used and no mean
term is stored (legacy v0.9.1 behaviour, exact reproducibility of
v0.9.1 fits).

## 5. Testing strategy

### 5.1 Smoke canary (tier 1, `tests/testthat/test-safety-floor.R`)

Runs on every `devtools::test()`, wall ~2 min on MPS.

Coverage:

1. **Simplex grid sanity.** `simplex_grid(0.05)` returns 231 rows,
   each summing to exactly 1, all entries ∈ [0, 1].
2. **Calibration invariant on validation.** Synthetic 3-trait
   dataset. After `fit_pigauto(..., safety_floor = TRUE)`, for
   every latent column, the recorded val loss at
   `(r_bm, r_gnn, r_mean)` is less than or equal to the val loss
   of the grand mean `(0, 0, 1)`.
3. **Pure-BM fallback.** `safety_floor = FALSE` on the same data
   reproduces v0.9.1 fits to within 1e-8 (no mean term).
4. **Backward compat.** Load a v0.9.1 fit saved under
   `inst/extdata/legacy_fit_v091.rds`, call `predict()`, confirm
   predictions are bit-identical to a freshly-run v0.9.1 predict
   on the same input.
5. **AVONET300 vertebrate regression.** Fit `safety_floor = TRUE`
   and `FALSE` on the bundled 300-species AVONET. Each continuous
   trait's safety-on RMSE must be within +2 % of safety-off RMSE;
   each discrete trait's safety-on accuracy must be within −1 pp
   of safety-off accuracy.
6. **Plants safety smoke.** Sample 300 species from the cached
   BIEN data. `safety_floor = TRUE` must produce every continuous
   trait RMSE ≤ 1.02 × grand-mean RMSE.
7. **MI mean-term propagation.** `multi_impute(m = 5,
   safety_floor = TRUE)` on the same plants 300-subset produces
   pooled SE that is not NaN on any trait.
8. **Degenerate binary MEAN clip.** Synthetic 2-trait dataset with
   one binary trait having class-1 frequency 0 in training. After
   `fit_pigauto(safety_floor = TRUE)`, `fit$mean_baseline_per_col`
   for that column is finite (`qlogis(0.01) = −4.60`) and the
   resulting prediction is a finite probability in `[0.01, 0.99]`.

### 5.2 Full canary (tier 2, `script/regress.R`)

Invoked as `Rscript script/regress.R` before a PR. Wall ~20 min on MPS.

For each of the five taxa — AVONET, PanTHERIA, FishBase, AmphiBIO,
BIEN — at `n = 1000` subset, seed 2026, `miss_frac = 0.30`,
`n_imputations = 10`:

1. Fit pigauto with `safety_floor = TRUE` and `safety_floor = FALSE`.
2. Compare per-trait RMSE / accuracy / conformal coverage /
   MC-dropout coverage.
3. **Pass criteria** (hard gate):
   - Continuous RMSE: `RMSE_on ≤ 1.02 × RMSE_off`
   - Discrete accuracy: `acc_on ≥ acc_off − 0.01`
   - Conformal coverage: `coverage_on ∈ [0.90, 1.00]`
   - **Plants safety claim (new):** `RMSE_on ≤ 1.02 × mean_baseline_RMSE`
     on every continuous BIEN trait.
4. Output: `script/regress_result.json` with per-bench per-trait
   pass/fail. Non-zero exit if any fail.

### 5.3 Deep canary (tier 3, manual, pre-release)

Same as tier 2 but at full scale (the existing overnight bench
commands). Run before tagging a release.

## 6. Risks and mitigations

### 6.1 Risk: vertebrate calibration degrades because the 3-D grid has coarser sampling of the `r_mean = 0` face

The 2-D grid `(r, 1-r)` at step 0.05 has 21 points on `r_mean = 0`.
The 3-D simplex grid at step 0.05 has 21 points on `r_mean = 0`
(the edge where `r_mean = 0`, running from `(1, 0, 0)` to `(0, 1, 0)`
with step 0.05). So sampling density on the old search space is
identical.

### 6.2 Risk: wall time increase from 21 → 231 calibration evaluations

Each evaluation is one dot product + sum over n_val cells (fast
vectorised R). At n_val ≈ 1000 the per-trait calibration cost rises
from ~20 ms to ~200 ms. For the AVONET full bench (p_latent ≈ 14
latent columns) total added wall ≈ 2.5 s against a ~9400 s fit — a
0.03 % overhead, far below measurement noise.

### 6.3 Risk: overfitting the calibration to validation

The calibration already uses hold-out validation; adding the mean
term expands the hypothesis class by one dimension. The val-set
split size (20 % of training cells) is large enough that the
sampling noise in the argmin is comparable to the step size (0.05).
Monitored by the +2 % test-set tolerance in the canary.

### 6.4 Risk: discrete-trait 0-1 loss is non-convex, argmin is ambiguous

Same concern as today's 1-D discrete calibration. When multiple
simplex points tie, we break ties by preferring lower `r_mean`
(conservative: favour the fitted model over the trivial mean when
the fitted model is equally good), then lower `r_gnn` (favour the
stable BM baseline over the GNN delta). This tie-break order is
documented in `calibrate_three_way()` and tested in smoke canary
item 2.

### 6.5 Risk: the 0.05 step is too coarse

The simplex `(r_bm, r_gnn, r_mean) = (0.475, 0.475, 0.05)` would
snap to `(0.5, 0.5, 0)` at step 0.05, losing a tiny amount of signal
on the exact optimum. If the canary at step 0.05 shows a vertebrate
regression, we will tighten to step 0.025 (861 points per trait,
still under 0.3 s per column at n_val = 1000). Out of scope for v1
spec; will be a follow-up if the canary flags it.

### 6.6 Risk: grand-mean leakage in multi-obs mode

In multi-obs mode, the grand mean must be computed from training
observations only — specifically, it must exclude held-out test
observations AND held-out validation observations during
calibration. Implemented by computing `MEAN_j` over
`X_scaled[training_split & !held_out, j]`, where `training_split`
is the post-aggregation species-level training index in single-obs
mode or the obs-level training index in multi-obs mode. Covered by
smoke canary item 7.

## 9. Open questions — all resolved

Every decision logged in the brainstorming transcript:

- **Rollout style:** flip default immediately with hard regression gate
  (Option 2).
- **Scope:** #1 (three-way mean-gate) only; #2 (phylogenetic-signal
  gate) and covariates ship as separate specs.
- **Safety claim strictness:** invariant by construction on validation;
  2 % sampling-noise tolerance on test.
- **Regression tolerances:** continuous RMSE ≤ +2 %, discrete accuracy
  ≥ −1 pp, conformal coverage ∈ [0.90, 1.00], plants continuous
  RMSE ≤ 1.02 × mean.
- **Canary tiers:** smoke (AVONET 300 + plants 300, ~2 min) on every
  commit, full (n = 1000 × 5 taxa, ~20 min) on every PR, deep (all
  taxa full scale) pre-release.
- **Weight constraint:** simplex, `r_BM + r_GNN + r_MEAN = 1`.
- **Grid step:** 0.05 (231 candidates per trait), widen to 0.025 as
  follow-up if canary flags vertebrate regression.
- **Backward compat:** old saved fits handled via `%||%` fallback;
  `fit$r_cal` continues to be written for downstream compatibility.
